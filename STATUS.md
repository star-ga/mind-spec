# Documentation Status

> **Last Updated:** 2026-07-02 · spec v1.0 · tracking compiler **v0.10.x** (reference implementation at [`star-ga/mind`](https://github.com/star-ga/mind)).
>
> **Backend architecture (pivot 2026-06-24):** The **native-ELF backend** (`src/native`) is the normative self-host target — determinism-by-construction (the native ELF is a pure function of the IR). The **MLIR-text backend** is a downstream-interchange and exotic-chip-reach backend; it is not the self-host path but is not dropped. "Target any chip" is implemented via a pluggable backend trait with commercial backends in `mind-runtime`.
>
> **Self-host status (v0.10.0, verified 2026-06-25):** The pure-MIND compiler front-end (`examples/mindc_mind/main.mind`) reproduces byte-for-byte against the Rust reference: (a) the `mic@1` IR-text bootstrap fixed point, (b) the `mic@3` canonical-binary-IR flip (CI gate `mic3_flip_smoke.py` 7/7), and (c) the **native x86-64/ELF emit of the entire seeded module** (21 stdlib modules + `main.mind`, 1 055 777 B) — the native-ELF self-host fixed point is closed. The G2 differential corpus (67 MATCH / 0 DIVERGE / 0 MIND_UNSUPPORTED / 12 RUST_ONLY) proves the front-end handles enum/extern/module/use/const/bare-expr constructs. **Update (2026-07-01):** both remaining items are now closed. The pure-MIND front-end self-computes the 32-byte `ir_trace_hash` PT_NOTE byte-identically at full scale (zero Rust-oracle bytes fed anywhere in the loop, the no-feed rung in `self_host_native_elf_smoke.py`), and the now-fully-redundant Rust `src/native` backend (2441 lines) has been deleted, after freezing its current output as permanent test fixtures (`examples/mindc_mind/testdata/native_elf_oracle/`). **Honest scope:** this means the **bootstrap/front-end self-hosts** — it does NOT mean the toolchain is Rust-independent. Full-chain Rust-independence of the whole toolchain (driver, `mindc build`/`test` orchestration, remaining Rust compiler phases) is roadmap, currently ~15% by surface.
>
> **Tensor status (v0.10.0):** Dense tensor literals, elementwise ops, and tensor-returning functions now compile and build to ELF (commits 9f807bd, cb43c90); int/Q16 results are byte-identical across the proven CPU substrates (cross_substrate 12/12); scalar `f32`/`f64` arithmetic (`+ − × ÷ √`) runs on the strict no-FMA path bit-exact and run-to-run bit-identical (cross-ISA verification in progress); cross-substrate byte-identity for `f32`/`f64` vector reductions is roadmap. Deterministic tensor intrinsics (`zeros`, `ones`, `matmul`, `softmax`, `transpose`, `randn`) and inter-function tensor-arg calls are Phase 11 roadmap. v0.9.0 landed narrow-int (`i32`/`u32`/`bool`) lowering in every context with cross-substrate shift-count determinism; v0.10.0 added running `f64` enum-payload fields and multi-field enum-payload variants (see [CHANGELOG](https://github.com/star-ga/mind/blob/main/CHANGELOG.md)). v0.8.0/v0.8.1 achieve byte-exact front-end compilation across the 66-fixture gap corpus (a fuzz sweep of arbitrary programs, not just `main.mind`), with **0 wrong-bytes** (the cardinal never-a-silent-miscompile invariant) and **0 fail-closed**; CI-gated by `mic3_flip_smoke.py` and `gap_corpus_smoke.py` (FLOOR 66/66). Release gate `#306` cleared (the `std.string`/`std.sha256`/`std.toml` byte-precise-store migration + keystone re-bless). The v0.7.1 execution surface still holds: high-level `std-surface` is the shipped default — ten desugaring "bricks", each keystone-verified 7/7 across substrates (integer/literal match, enum-discriminant match, `Option`/payload match, string literals, struct field-write, nested-struct + borrow field access, method-as-field accessors `s.len()`, and method-with-args UFCS `v.push(x)` → `vec_push(v, x)`); cross-substrate Q16.16 byte-identity preserved; unpromoted constructs (by-value tuple/aggregate returns, `region { }` interior) re-gated behind `std-surface-experimental` and fail loud by default; `--no-default-features` restores low-level-only subset. Prior v0.7.0 work: Mindcraft RFC 0007 fully shipped; RFC 0008 all 7/7 phases; RFC 0010 Phases A/B/C + J-A/J-B; RFC 0011 Phase A; RFC 0012 tensor-native syntax; 13 stdlib modules. See [`star-ga/mind CHANGELOG`](https://github.com/star-ga/mind/blob/main/CHANGELOG.md) for full per-version record.

> **Crypto & protocol primitives status (v0.10.x, 2026-07-02):** the reference implementation ships pure-MIND, RFC/NIST-KAT-verified primitive modules: AES-128-GCM, SHA-256, HKDF, Keccak/SHA-3+SHAKE (FIPS 202), X25519, RSA-PSS-SHA256 (verify), ECDSA-P256 (verify), ML-KEM-768 (FIPS 203, post-quantum), X.509/DER parse+verify (single-certificate), TLS 1.3 key schedule / record layer / Finished MAC / handshake-crypto orchestration (validated by RFC 8448 trace replay), HPACK (RFC 7541, decode-only), and HTTP/2 framing (RFC 9113). **Honest scope: this is a verified primitive library, NOT a working TLS client/server** — no socket-level handshake, no certificate-chain validation (no path building / trust anchors / revocation). Correctness-first: KAT-bit-exact and deterministic, with no constant-time/side-channel claim and no performance claim. Live TLS client, chain validation, more suites (ChaCha20-Poly1305, AES-256-GCM, SHA-384/512, Ed25519, ML-DSA), and HTTP/3+QUIC are roadmap. Normative scope statement: [`spec/v1.0/stdlib.md`](spec/v1.0/stdlib.md#cryptographic--protocol-primitive-modules-informative--implementation-status).
>
> **Evidence-chain status:** the chain is **emitted/embedded, not yet cryptographically signed** — Ed25519 signing (RFC 0016 Phase C) is the next milestone. Never describe it as a "signed evidence chain".

This status page provides a quick view of the readiness of key documentation areas within the MIND specification. Use it to coordinate work, plan reviews, and spot sections that still need expansion.

## Mindcraft (RFC 0007) — Fully Shipped

All six phases and the MINDCRAFT-001 keystone landed in mindc v0.6.8.
The self-hosting credibility ladder milestones to date: the language
self-hosts (v0.6.0 apex), the compiler **front-end** self-hosts (v0.6.1
fixed-point, extended to the native-ELF gate in v0.10.0), and the
Mindcraft developer tooling (`fmt`/`lint`/`check`) is written in pure
MIND (v0.6.8). This does **not** make the toolchain as a whole
self-hosted or Rust-independent — that is roadmap.

- Phase 1: `MindcraftConfig` manifest types in `Mind.toml` (`6526029`)
- Phase 2A: `mindc fmt` with `--check` / `--diff` / `--stdin` / `--fix` (`6e36fa3`)
- Phase 3: Lint rule infrastructure + glob `overrides` + `RuleRegistry` (`ccbaba9`)
- Phase 4: 5 named lint rules (`q16_overflow`, `unused_import`, `naming_convention`, `shadowing`, `trailing_whitespace`) (`5ff5367`)
- Phase 5: `mindc check` project driver — VCS-aware, JSON + LSP reporters (`1442a31`)
- Phase 6: `--fix` pipeline + CI integration + `.github/workflows/mindcraft.yml` (`15f9960`)
- MINDCRAFT-001 keystone: `pub` keyword preserved through AST and formatter (`1d988bd`)

Canonical spec: [`docs/rfcs/0007-mindcraft.md`](https://github.com/star-ga/mind/blob/main/docs/rfcs/0007-mindcraft.md) in `star-ga/mind`.
